# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

Open Azitfh — Korean community site. Static HTML/CSS/JS, no build step. Open `index.html` directly in browser. Auth via Supabase (credentials hardcoded in `supabase-config.js`). All data (posts, categories, notices) stored in localStorage.

## Data

| localStorage key | schema |
|---|---|
| `jnaver_posts` | `{id, title, content, category, authorId, authorNickname, createdAt, views}[]` |
| `jnaver_categories` | `string[]` |
| `jnaver_notices` | `{id, title, date}[]` |
| `jnaver_theme` | `'dark' \| 'light'` |

## Pages → init functions

Each HTML file calls `initDarkMode()` + one page init at bottom of `<body>`.

`index.html` → `initIndex()` · `post-write.html` → `initPostWrite()` · `post-detail.html` → `initPostDetail()` · `dashboard.html` → `initDashboard()` · `login.html` → `initLogin()` · `signup.html` → `initSignup()`

## app.js structure

1. Dark mode, toast, error helpers
2. Supabase auth wrappers: `authSignUp` `authSignIn` `authSignOut` `getSession`
3. `updateNav(session)` — shows/hides nav links based on login state
4. Notices: `getNotices` `saveNotices` `renderNotices` `initNotices` — rendered as `.news-card` grid
5. Categories: `getCategories` `saveCategories` `renderCategoryChips` — `_selectedCat` global tracks active filter
6. Posts: `getPosts` `savePosts` `getPost` `deletePost` `incrementViews` `renderPosts`
7. Page inits at bottom

## Rules

- **Login = admin.** No separate role system. Session presence controls write access.
- All user input → `escapeHTML()` before DOM insertion.
- Categories must exist before writing a post (dashboard adds them).
- Logo: `logo.png` in root. Generated by `hero-gen/generate-logo.js` (Gemini Imagen 4).
- `hero-gen/` is gitignored. Contains `generate-hero.js`, `generate-logo.js`, `run.bat`, `config.json` (API key). Images saved as `hero-N.png` / `logo-N.png` with auto-incrementing number.
